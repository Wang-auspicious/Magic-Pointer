# 距离桌面 AGI 的版本差距记录

本文件用于记录每一版完成后，距离“桌面 AGI / Magic Pointer 完整愿景”还差什么。原则：每版都要可运行、可演示、可评估，不用愿景掩盖实际缺口。

## 桌面 AGI 目标定义（工作版）

桌面 AGI 在本项目中的含义不是通用智能，而是一个能在用户授权下跨应用完成真实桌面任务的系统：

```text
理解当前屏幕与用户指向
  -> 把屏幕内容注册成对象
  -> 理解短指令和指代关系
  -> 制定多步计划
  -> 操作应用、网页、文件和剪贴板
  -> 校验结果
  -> 在失败时解释原因并请求确认
```

## v0.0.1 / MVP0：全局框选截图问答

### 已完成

- 全局热键 `Ctrl + Alt + M` 唤起。
- 鼠标框选任意屏幕区域。
- 保存截图为本地对象。
- 输入短指令并调用多模态模型。
- 悬浮结果窗展示回答。
- 本地 JSONL 对象日志。

### 已经接近桌面 AGI 的部分

- 用户可以用“指向/框选”代替复制粘贴和长 prompt。
- 系统已经有最小对象概念：`screen_region`。
- 已经具备 “屏幕局部 -> AI 理解 -> 结果展示” 的闭环。

### 距离桌面 AGI 的差距

1. **对象层不足**
   - 现在只有截图对象，没有 DOM、Accessibility、OCR 文本、剪贴板文本、文件对象。
   - `this/that/group/destination` 还没有真正实现，只记录了 latest object。

2. **执行层缺失**
   - 只能回答，不能写回、点击、打开地图、创建日历、生成文件。
   - 没有安全权限分级，也没有执行前确认机制。

3. **校验层缺失**
   - 不能验证粘贴是否成功、目标是否正确、任务是否完成。

4. **规划层缺失**
   - 不能把“把这里总结后放到那里”拆成读取、生成、定位、写入、校验。

5. **屏幕理解还粗糙**
   - 没有 OCR 兜底。
   - 没有 UI 元素检测模型，例如 OmniParser。
   - 没有区分文本、图片、表格、代码、商品、日期等对象类型。

6. **浏览器能力缺失**
   - 没有 Chrome 插件。
   - 不能读取 DOM、链接、表格结构、图片 URL、选中文本和输入框位置。

7. **产品体验不足**
   - 没有系统托盘。
   - 没有应用白名单/黑名单。
   - 没有勿扰、频率限制、触发方式配置。
   - UI 只是工程原型。

## 下一版建议：v0.1.0 / MVP1 对象注册与多对象指代

目标：让产品从“截图问答工具”变成“指针对象工具”。

优先做：

- 对象面板：显示最近选择的对象。
- 对象别名：this / that / group / destination。
- 连续框选多个对象。
- “比较这个和那个”。
- “合并这些”。
- 对每个对象做 OCR 文本缓存。
- 对象日志升级为 SQLite 或结构化 JSON。

完成 v0.1.0 后，再进入 v0.2.0：文本选区读取、改写、覆盖和写回校验。

## v0.1.0-beta / MVP1-beta: Hidden task context and session-scoped references

### Completed

- Replaced the always-visible historical object panel with a lightweight context bar.
- Added `TaskContextStore` with current task, previous task, messages, and object IDs.
- AI reference context is now scoped to the current task, not global object history.
- `this` means the current selection.
- `that` means the previous object in the current task.
- `group` means current task objects plus this.
- A new task starts automatically after 30 minutes of inactivity.
- Users can explicitly start a new task or restore the previous task.
- The context drawer is hidden by default and only expands on demand.

### Closer to desktop AGI

- The system now has a practical task boundary without forcing a visible chat-history product shell.
- Short one-off tasks stay lightweight.
- Longer tasks can continue through the task context layer.
- Old screenshots are preserved as logs but do not silently affect new reasoning.

### Remaining distance

1. There is still no full long-conversation UI or task browser.
2. Destination is still only reserved, not implemented.
3. There is no OCR/UIA/DOM/accessibility object tree yet.
4. The app still answers only; it cannot write back or execute cross-app actions.
5. There is no execution confirmation, permission model, or result verification.

## v0.1.0-gamma / MVP1-gamma: Task-scoped destination

### Completed

- Added a `DESTINATION` alias to the current task context.
- Users can set the current selection as destination from the prompt window.
- Users can clear the current task destination.
- The context bar shows the destination state without opening a heavy history UI.
- Model context now distinguishes THIS, THAT, GROUP, and DESTINATION.
- Destination-like prompts attach the explicit destination reference image when available.

### Closer to desktop AGI

- The system can now represent "put it there" style references without relying on global history.
- Destination is scoped to the active task, so unrelated old screenshots do not pollute target selection.
- This prepares for later write-back/execution layers.

### Remaining distance

1. Destination is understood but not acted on; there is still no automatic write-back.
2. No UIA/OCR/DOM target extraction yet, so destination is still a screenshot object.
3. No permission/confirmation model for performing actions.
4. No execution verification after writing or pasting.

## v0.1.0-delta / MVP1-delta: Pointer command bar, not chat window

### Completed

- Replaced the default large prompt/result chat window with a compact command bar near the selected region.
- Added quick actions for explain, compare, destination, details, and continue selection.
- Results are shown as a short action-card style response.
- Full details are available only on demand.
- Task context remains hidden by default but still drives THIS/THAT/GROUP/DESTINATION semantics.

### Closer to desktop AGI

- Interaction is now pointer-native: select something, give a short command, get a small result card.
- The UI no longer encourages long chatbot-style conversations for every screenshot.
- The design better matches the Magic Pointer idea: motion/selection plus terse intent, not a separate AI app.

### Remaining distance

1. No voice input yet; commands are still typed or button-triggered.
2. No proactive contextual suggestions yet.
3. No true write-back/execution; destination is understood but not acted on.
4. No system tray/native shell integration yet.

## v0.1.0-epsilon / MVP1-epsilon: Voice entry and contextual suggestions

### Completed

- Added a `??` button to the pointer command bar.
- Voice input uses Windows built-in dictation (`Win + H`) instead of adding a new speech dependency.
- Added context-aware default commands for explain/compare/destination flows.
- Added `???` as a quick action tied to current-task DESTINATION.

### Closer to desktop AGI

- Interaction is now closer to pointer + short natural instruction.
- Users can speak a short intent instead of typing a prompt.
- The command bar begins to suggest likely next actions based on current task context.

### Remaining distance

1. Voice is delegated to Windows dictation, not an integrated always-on speech pipeline.
2. Suggestions are rule-based, not learned/proactive.
3. No execution/write-back yet.
4. No system tray/native shell integration yet.
